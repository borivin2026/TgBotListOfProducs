# Техническое задание (ТЗ): Telegram-бот "Smart Shopping List"

## 1. Общее описание
Бот предназначен для автоматизации создания списков покупок с помощью голоса, интеллектуальной обработки текста и оцифровки бумажных чеков. Бот ориентирован на удобство использования "на ходу" и хранение истории покупок.

## 2. Функциональные требования

### 2.1. Обработка голосовых сообщений
- **Действие:** Пользователь отправляет голосовое сообщение со списком товаров (например: "Купи молоко, две буханки хлеба и десяток яиц").
- **Реализация:**
  1. Бот скачивает `.ogg` файл.
  2. Передает его в **Deepgram API** для высокоточного распознавания речи (STT).
  3. Полученный текст передается в **Gemini 1.5 Flash**.
  4. Gemini возвращает структурированный список товаров в формате JSON.

### 2.2. Ведение активного списка и уточнения
- **Действие:** Пользователь может дополнять или изменять текущий список голосом или текстом (например: "Ой, еще добавь масло" или "Хлеба не два, а один").
- **Реализация:**
  - Использование **FSM (Finite State Machine)** для хранения ID активного списка.
  - При каждом новом вводе бот передает Gemini *текущий список* + *новое сообщение*.
  - Модель возвращает обновленную версию списка (merge logic).

### 2.3. Экспорт в CSV
- **Действие:** В каждом ответе бота прикрепляется файл для скачивания.
- **Реализация:** Генерация CSV-строки на лету с использованием библиотеки `csv`. Колонки: `Товар`, `Количество`, `Дата`.

### 2.4. Финализация списка
- **Действие:** Кнопка "✅ Подтвердить список".
- **Реализация:** Бот переводит статус списка в БД из `active` в `completed`, очищает состояние пользователя и прекращает режим редактирования.

### 2.5. Распознавание чеков
- **Действие:** Пользователь хочет оцифровать чек. Для этого он должен вызвать команду `/receipt` (или нажать кнопку "Сканировать чек"), после чего отправить фото. Просто отправка фото без команды не должна запускать процесс распознавания.
- **Реализация:**
  1. Бот переходит в состояние `waiting_for_receipt`.
  2. Пользователь может отправить фото ИЛИ нажать кнопку "Отмена" (или отправить команду `/cancel`), чтобы вернуться в режим обычного списка/голосовых сообщений.
  3. Если отправлено фото: Gemini Vision -> Сохранение -> Возврат в основное состояние.

### 2.7. Доменные сущности (Domain Entities)
Для обеспечения чистоты кода и типизации выделяются следующие сущности:
- **User**: Информация о пользователе Telegram.
- **ShoppingList**: Группа товаров, объединенных одной сессией.
- **ListItem**: Конкретный товар (название, количество, статус).
- **Receipt**: Результат оцифровки чека (список позиций, дата, сумма).

## 3. Технические требования

### 3.1. Качество кода
- **Типизация:** Обязательное использование Type Hints (`typing`) для всех функций и классов.
- **Модульность:** Разделение на независимые модули (API, DB, UI, Models).
- **Документирование:** Комментарии в сложных блоках бизнес-логики (особенно в части слияния списков через Gemini).
- **Core:** Python 3.10+, `aiogram 3.x`.
- **APIs:** Deepgram SDK, Google Generative AI SDK.
- **DB:** `aiosqlite` (асинхронная работа с SQLite).
- **Env:** `python-dotenv` для хранения ключей.

### 3.2. Логирование
- Логирование всех запросов к API, ошибок и действий пользователей в файл `bot.log`.
- Уровень логирования: `INFO` для действий, `ERROR` для сбоев.

### 3.3. Тестирование и Деплой
- **Локально:** Long Polling.
- **Продакшн:** Webhooks на **PythonAnywhere**.
- **Архитектура Webhook:** Использование легковесного сервера (Flask или FastAPI) для приема POST-запросов от Telegram и передачи их в `aiogram`.

## 4. Алгоритм работы (Flow)

1. **Start:** `/start` -> Регистрация пользователя в БД.
2. **Input:** Голос/Текст -> Deepgram -> Gemini (Extract) -> Update DB -> Send List Message + Inline Button + CSV.
3. **Refinement:** Новый ввод -> Gemini (Merge) -> Update DB -> Send Updated List.
4. **Receipt:** Photo -> Gemini Vision -> Save to `receipts` -> Confirmation message.

## 5. Способ реализации (Архитектура)
Проект будет разбит на модули:
- `handlers/`: Обработка команд, голоса, фото и текста.
- `services/`: Модули `deepgram_service.py`, `gemini_service.py`, `csv_service.py`.
- `scripts/`: Скрипты для изолированной проверки API (например, `test_gemini.py`).
- `db/`: Менеджер соединений и SQL-запросы.
- `config.py`: Загрузка переменных окружения и настроек логов.
