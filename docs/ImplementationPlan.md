# План реализации Telegram-бота "Список покупок"

## 🛠 Технологический стек
- **Язык:** Python 3.10+
- **Библиотека Telegram:** `aiogram 3.27.0+` (асинхронность, мощная система FSM и Webhooks)
- **Распознавание речи:** Deepgram API (SDK v7+, модель `nova-3`)
- **Интеллектуальная обработка:** Google GenAI SDK (модель `gemini-2.0-flash`)
- **База данных:** SQLite (через `aiosqlite`)
- **Формат экспорта:** CSV (стандарт RFC 4180)
- **Развертывание:** PythonAnywhere (через Flask/FastAPI адаптер для Webhooks)

---

## 📋 Этапы разработки

### Этап 1: Подготовка проекта
- [ ] Инициализация Git-репозитория.
- [ ] Настройка структуры проекта:
  - `bot/` (логика бота: хендлеры, мидлвари)
  - `database/` (схема БД, запросы)
  - `services/` (Deepgram, Gemini, CSV Utils)
  - `scripts/` (простые скрипты для проверки работы API)
  - `utils/` (логирование, конфиг)
- [ ] Установка зависимостей (`aiogram`, `google-genai`, `deepgram-sdk`, `python-dotenv`, `aiosqlite`, `pydantic`).
- [ ] **Domain Modeling:** Создание `models/` с классами сущностей (User, ListItem, ShoppingList, ReceiptItem) с использованием `dataclasses` или `pydantic` для строгой типизации.

### Этап 2: Слой данных (SQLite)
- [ ] Проектирование схемы БД:
  - `users`: ID, username.
  - `shopping_lists`: ID, user_id, статус (активен/завершен), дата создания.
  - `list_items`: ID, list_id, название, количество, примечание.
  - `receipt_items`: ID, user_id, название, цена, дата (из чеков).
- [ ] Реализация асинхронных методов работы с БД.

### Этап 3: Интеграция внешних сервисов
- [ ] **Service: Deepgram**
  - Загрузка голосового файла из TG -> Отправка в Deepgram -> Получение текста.
- [ ] **Service: Gemini (Text)**
  - Системный промпт для извлечения товаров в JSON-формате.
  - Логика "merge" (слияние) текущего списка с новыми уточнениями пользователя.
- [ ] **Service: Gemini (Vision)**
  - Промпт для распознавания табличных данных на фото чеков.
- [ ] **Service: CSV Exporter**
  - Генерация файла на основе текущего состояния списка.

### Этап 4: Разработка интерфейса бота (UI/UX)
- [ ] Основное меню и обработка команд (`/start`).
- [ ] Обработка Voice-сообщений:
  - Состояние ожидания -> Процессинг -> Вывод списка.
- [ ] Кнопки (Inline Keyboards):
  - "✅ Принять как окончательный список"
- [ ] Обработка уточнений:
  - Если активен режим составления списка, любой текст или голос считается уточнением.
- [ ] Отправка CSV файла в каждом сообщении с обновленным списком.
- [ ] Обработка фото (Чеки):
  - Команда `/receipt` -> Состояние `waiting_for_receipt`.
  - Кнопка "Отмена" / Команда `/cancel` -> Сброс состояния, возврат к списку.
  - В состоянии ожидания: получение фото -> Gemini Vision -> Сохранение -> Выход из состояния.

### Этап 5: Логирование и диагностика
- [ ] Настройка записи логов в файл `bot.log`.
- [ ] Логирование критических ошибок и действий пользователя.

### Этап 6: Деплой и Webhooks
- [ ] Реализация переключателя `polling` vs `webhook`.
- [ ] Создание входной точки (app.py) для PythonAnywhere.

## ❓ Вопросы для уточнения

1. **Формат CSV:** Какие именно колонки нужны? (например: `Товар`, `Количество`, `Примечание`).
2. **Дубликаты в списке:** Если пользователь сказал "хлеб" дважды, мы суммируем количество или создаем две записи?
3. **Хранение аудио:** Нужно ли сохранять сами голосовые сообщения локально или удалять после распознавания?
4. **Безопасность:** Нужен ли доступ по белому списку ID пользователей или бот публичный?
